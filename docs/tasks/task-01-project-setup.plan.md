## File inventory (deliverables)

### Root config
- `package.json` — pnpm, Next 15, Payload 3, TanStack Query, next-intl, dev tooling.
- `tsconfig.json` — `strict: true`, `noUncheckedIndexedAccess: true`, path aliases.
- `next.config.ts` — Payload plugin wrap + next-intl plugin wrap.
- `tailwind.config.ts` — content globs: `app/`, `components/`, `payload/`. Server-only safe.
- `postcss.config.mjs` — Tailwind + autoprefixer.
- `.eslintrc.json` — `next/core-web-vitals` + `@typescript-eslint` + `jsx-a11y`.
- `.prettierrc.json` + `.prettierignore`.
- `.gitignore` (covers .next, node_modules, .env, .turbo, payload-types.ts).
- `.nvmrc` — Node 20 LTS.
- `.env.example` — every variable expected (DB, Payload secret, app URL, etc.).
- `README.md` — ≤ 20 lines, dev workflow only.

### App Router skeleton (Next + public site)
- `app/layout.tsx` — root layout, `next/font` (Inter), html lang, `<Providers>` wrap.
- `app/page.tsx` — placeholder home (no marketing content).
- `app/providers.tsx` — `'use client'`, TanStack Query, `staleTime: 30_000`.
- `app/globals.css` — `@tailwind base/components/utilities`.

### i18n skeleton (next-intl)
- `i18n/request.ts` — `getRequestConfig` stub returning empty messages. **Task-29 fills this.**
  *Note:* No middleware, no `[locale]` segment yet — the skeleton is just enough for next-intl's plugin to be wirable.

### Payload 3 (route group)
- `payload.config.ts` — Postgres adapter on `payload` schema, lexical editor, empty `collections: []`.
- `app/(payload)/layout.tsx` — Payload's own layout (escapes the public root layout).
- `app/(payload)/admin/[[...segments]]/page.tsx` + `not-found.tsx` — admin shell.
- `app/(payload)/api/[...slug]/route.ts` — REST endpoints.
- `app/(payload)/api/graphql/route.ts` + `graphql-playground/route.ts` — GraphQL.

### Type seed
- `next-env.d.ts` is auto-generated by Next on first build — not committed by hand.
- `payload-types.ts` is generated by `pnpm payload generate:types` — gitignored stub.
